
Q1: Why might be consider using git?

Answer: Git is very helpful for when we need to collaborate while coding. It maintains a history of changes made to the code, you can isolate code features by branching off of the main version, and git can also support a peer review process for our code.

Q3: What is the difference between merging and rebasing?

Answer: merging applies the changes made on one branch to another branch we are merging with. Rebasing moves the entire branch you are rebasing to the tip of the other branch, rewriting the project history. Instead of keeping two separate branches like with merge, rebase produces one longer branch, producing a cleaner project history.
